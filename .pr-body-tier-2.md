## Summary

Tier 2 of the end-to-end review fix-pack. Three workflow features the review surfaced as the biggest day-to-day pains for the PM running 10–20 active builds.

**Stacked on `claude/tier-1-fixes`** — set the merge base to that branch (or merge tier-1 first so this rebases onto main cleanly).

## 1 · Schedule: multi-select + bulk actions

The schedule list view now lets a PM tick rows across work items, child to-dos, and unlinked to-dos, then apply one action to all of them.

- **Shift dates by ±N days** with full predecessor + anchored-children cascade
- **Set status** (one of the four canonical states)
- **Delete** — refuses if any selected item is a predecessor of an unselected one, surfacing the blocking IDs so the PM can either include them or use the single-item delete + reassignment flow

Server actions all live in `app/actions/schedule.ts`, return `{ ok, skipped: [{id, reason}] }` so the UI can show partial-failure detail, are capped at 500 ids per call, and are scoped by project_id for RLS containment.

UI: `components/schedule/bulk-actions-bar.tsx` is a sticky floating bar that appears whenever ≥1 item is selected. Selected rows get a subtle brand-50 wash.

## 2 · Companies: sub trade tagging

A company can do more than one trade. The existing `trade_category` text column captured one tag at a time. Promoted to a real many-to-many.

- Migration **0026** (applied to HH-ProjectManager): `company_trades(company_id, trade)` PK junction, free-text but normalized (1–60 chars, lower-cased, trimmed via CHECK constraint), indexed on `(trade)` for "which subs do framing" lookups. Backfilled from the comma-split `trade_category` of the 3 currently-tagged companies.
- `saveCompany` accepts `trades: string[]`, dedupes + sorts, replaces the row set on every save. Cap at 20 trades. `trade_category` is kept in sync as a back-compat mirror of the first trade.
- `components/companies/trade-chips-editor.tsx`: chip input with Enter/comma to commit, Backspace-on-empty to pop the last chip, suggestion row sourced from the global pool so staff pick the existing "framing" instead of accidentally creating "framers".
- Companies page toolbar gains a trade-filter chip row. Table cells render trades as small brand-100 chips. Search now matches trades too.

## 3 · SMS to subs

Wires the existing Quo integration into the two paths where a PM expects a sub to hear back without thinking about it:

- **Schedule assignment** — `notifyScheduleAssignees` now also texts every company-typed assignee with a phone on file. Mirrors the email-on-assign path. Email + SMS fan out via `Promise.allSettled` so a slow Quo can't delay the schedule save.
- **Daily log subs-on-site** — when a sub is *freshly* added to a daily log's `subs_on_site` list (diffed against the prior list, so a PM editing notes doesn't re-text the crew), they get a short text confirming the on-site entry with date + project label.

Both paths are silent no-ops if `QUO_API_KEY` / `QUO_FROM_NUMBER` aren't set — `sendQuoSms` already degrades gracefully in dev/preview.
