## 1 · Pricing: retainage tracking + lien waivers

Two construction-finance basics the original review flagged as missing.

**Migration 0027 (applied to HH-ProjectManager):**
- `projects.retainage_percent numeric(5,2) NOT NULL default 0`, CHECK 0–25. Stored as % so a project that drops retainage to 0 on closeout doesn't need a separate amount column rewritten.
- `project_payments.lien_waiver_received boolean default false` + `lien_waiver_reference text` (check #, DocuSign envelope, "see folder", whatever proves it).
- Partial index `idx_pp_waiver_missing` for "active payments missing a waiver" so the upcoming aging report has a fast path.

**Pricing UI** (`app/(app)/projects/[id]/pricing/*`):
- "Balance due" summary card relabels to "Balance due (after X% retainage)" when retainage > 0; thin info row beneath shows retainage-held + balance-before-retainage.
- Same row surfaces a "N payments missing a lien waiver" warning when any exist (amber dot for at-a-glance scanning).
- Page query now hides soft-deleted payments from staff too (Tier 1 left them visible) — deletions survive in `payment_audit`.
- `PaymentDialog` gets a "Lien waiver received" checkbox + reference text field that appears only when checked. Reference is cleared server-side if the box is unchecked so a leftover string can't pretend to mean "received".

**EditProjectDialog** gets a "Retainage %" field bounded 0–25 client-side; Zod re-checks on the server.

## 2 · Projects: portfolio health dashboard

Replaces the bare `/projects` table with the at-a-glance ops view the review called for. Lets a PM running 5–30 active jobs answer "which of them needs me today" without clicking into each one.

**Top stats band** (4 cards, only shown when at least one project is active or pre-construction):
- **On track**: active projects with no delayed and no past-due items
- **Behind**: active projects with at least one delayed or past-due item
- **Contract value**: sum of contract_price across every visible project
- **Cost growth**: sum of approved cost_deltas, plus % of total contract

**Per-row enhancements** (visible at `xl+` for progress/health, `lg+` for financials, so mobile still works):
- **Progress bar**: schedule_items completed / total, color-graded amber → brand → success at the 60% / 100% thresholds
- **Schedule health**: loudest signal first — "N delayed" (red) > "N past due" (amber) > "N in progress" (brand) > "on track" (muted). Closed and cancelled projects show "closed" so they don't dominate the picture.
- **Changes column**: approved cost-delta total, signed and colored (amber for net add, success for net reduction)

**Data layout**: two extra queries against `schedule_items` and `decisions`, both predicated on the visible project ids. Single pass to roll up per project; even at 30 projects × 200 items = 6k rows this is well inside the SSR round-trip budget. No N+1.
